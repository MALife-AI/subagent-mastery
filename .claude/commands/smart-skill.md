# /smart-skill

사용자의 자연어 지시를 받아 **사용 가능한 스킬을 최대한 활용**하도록 query를 재작성하고, 사용할 스킬을 확인받은 뒤 실행합니다.

## Goal

사용자가 스킬 이름을 몰라도 자연어로 지시하면, 최적의 스킬 조합을 자동으로 매칭하고 실행합니다.

## 실행 순서

### Step 1: 스킬 카탈로그 로드

현재 사용 가능한 스킬 목록을 확인합니다:

| 스킬 | 트리거 키워드 | 설명 |
|------|-------------|------|
| `banner-design` | 배너, 커버, 헤더, 썸네일, 광고 이미지 | 소셜/광고/웹 배너 디자인 |
| `brand` | 브랜드, 보이스, 아이덴티티, 메시징, 톤앤매너 | 브랜드 전략 및 비주얼 아이덴티티 |
| `composition-patterns` | 컴포넌트, 합성 패턴, compound, render props | React 컴포지션 패턴 |
| `design` | 디자인, 로고, CIP, 아이콘, 소셜 이미지 | 종합 디자인 (로고/CIP/슬라이드/배너/아이콘) |
| `design-system` | 토큰, 디자인 시스템, CSS 변수, 컴포넌트 스펙 | 디자인 토큰 아키텍처 |
| `mirae-asset-design` | 미래에셋, 금융 UI, 리포트, 차트 | 미래에셋 공식 디자인 가이드라인 |
| `react-best-practices` | React 성능, Next.js, 최적화, SSR, RSC | React/Next.js 성능 최적화 |
| `react-native-skills` | React Native, Expo, 모바일, 앱 | RN/Expo 모바일 앱 최적화 |
| `slides` | 슬라이드, 프레젠테이션, 발표, PPT, 피치덱 | HTML 프레젠테이션 생성 |

| `web-design-guidelines` | 웹 디자인 리뷰, 접근성 감사, UI 검토 | 웹 인터페이스 가이드라인 리뷰 |
| `everything-claude-code` | ECC, 개발 컨벤션 | ECC 프로젝트 개발 컨벤션 |

| `production-design` | 인터페이스 설계, 폴리싱, audit, critique, 색상, AI slop, 프로덕션 | 프로덕션급 프론트엔드 설계 (24개 커맨드) |

| `design-quality` | 안티패턴, AI 제네릭, 레이아웃, 모션 강도, 밀도, bento | 메트릭 기반 디자인 (AI Tells 금지) |

또한 시스템에 등록된 외부 스킬도 포함합니다:

| 스킬 | 트리거 키워드 | 설명 |
|------|-------------|------|

| `code-review` | 코드 리뷰, PR 리뷰 | Pull Request 코드 리뷰 |

| `commit` | 커밋 | Git 커밋 생성 |

### Step 2: Query 분석 및 스킬 매칭

사용자의 지시를 분석하여 다음을 수행합니다:

1. **의도 파악**: 사용자가 무엇을 하려는지 핵심 의도를 추출합니다
2. **키워드 매칭**: 지시에 포함된 키워드를 스킬 트리거와 매칭합니다
3. **스킬 우선순위 결정**: 가장 관련성 높은 스킬을 1순위로, 보조 스킬을 2순위로 정렬합니다
4. **Query 재작성**: 매칭된 스킬의 강점을 최대한 활용하도록 원본 지시를 구체적으로 재작성합니다

### Step 3: 사용자 확인 (필수)

**반드시** 다음 형식으로 사용자에게 확인을 요청합니다.
사용자가 비개발자일 수 있으므로, 각 스킬이 **실제로 무슨 일을 하는지 비유나 쉬운 말로** 설명합니다.
기술 용어를 쓸 때는 반드시 괄호 안에 쉬운 설명을 병기합니다.

```
📋 스킬 매칭 결과

원본 지시: {사용자의 원본 지시}

🔧 사용할 스킬:

  1순위: {스킬명}
     🎯 하는 일: {이 스킬이 구체적으로 무엇을 해주는지 비개발자도 이해할 수 있게 1~2문장}
     📌 왜 필요한가: {이 작업에서 이 스킬이 왜 쓰이는지 이유}

  2순위: {스킬명} (있는 경우)
     🎯 하는 일: {설명}
     📌 왜 필요한가: {이유}

🤝 스킬 협업 방식:
  {여러 스킬이 어떻게 함께 작동하는지 쉬운 비유로 설명}
  예: "{스킬A}가 설계도를 그리면, {스킬B}가 실제 부품을 조립합니다"

📝 재작성된 Query:
  {스킬에 최적화된 구체적 지시}

이대로 진행할까요? (Y/수정사항)
```

### 쉬운 설명 작성 가이드라인

스킬별 역할을 설명할 때 다음 비유를 활용합니다:

| 스킬 | 쉬운 비유 |
|------|----------|
| `ui-ux-pro-max` | **인테리어 디자이너** — 화면의 전체 분위기, 배치, 색상 조합을 잡아주는 역할 |
| `ui-styling` | **가구 제작자** — 실제 버튼, 표, 입력창 같은 화면 부품을 예쁘게 만들어주는 역할 |
| `design-system` | **인테리어 규격서** — 모든 화면에서 같은 색상, 같은 간격, 같은 글꼴을 쓰도록 규칙을 정하는 역할 |
| `design` | **종합 디자인 스튜디오** — 로고, 명함, 발표자료, 배너 등 시각물을 한곳에서 만드는 역할 |
| `mirae-asset-design` | **미래에셋 브랜드 가이드북** — 미래에셋 공식 색상, 글꼴, 로고 규정을 적용해주는 역할 |
| `brand` | **브랜드 컨설턴트** — "우리 브랜드가 어떤 말투와 이미지로 고객에게 보여야 하는지" 정리하는 역할 |
| `slides` | **발표자료 전문 디자이너** — 차트가 포함된 깔끔한 프레젠테이션을 만들어주는 역할 |
| `banner-design` | **광고 배너 디자이너** — SNS, 웹사이트, 광고용 이미지 배너를 만들어주는 역할 |
| `frontend-design` | **웹페이지 시공사** — 디자인을 실제 작동하는 웹페이지로 만들어주는 역할 |
| `react-best-practices` | **웹 성능 튜너** — 웹페이지가 빠르게 뜨고 부드럽게 작동하도록 최적화하는 역할 |
| `react-native-skills` | **모바일 앱 전문가** — 스마트폰 앱을 만들 때 필요한 노하우를 제공하는 역할 |
| `composition-patterns` | **레고 설계사** — 화면 부품을 레고처럼 조립·재사용할 수 있게 구조를 잡는 역할 |
| `web-design-guidelines` | **웹 품질 검수관** — 만들어진 화면이 표준과 접근성 기준을 충족하는지 검사하는 역할 |
| `deploy-to-vercel` | **출판사** — 만든 웹사이트를 인터넷에 공개해서 누구나 볼 수 있게 하는 역할 |
| `claude-api` | **AI 연결 기사** — Claude AI를 내 앱에 연결해서 AI 기능을 넣어주는 역할 |
| `code-review` | **코드 교정 편집자** — 작성된 코드에 실수나 개선점이 없는지 검토하는 역할 |
| `feature-dev` | **기능 설계사** — 새로운 기능을 체계적으로 분석하고 구현 계획을 세우는 역할 |
| `commit` | **변경 기록 담당자** — 작업 내용을 깔끔하게 저장하고 기록하는 역할 |
| `simplify` | **코드 정리 전문가** — 작성된 코드를 더 깔끔하고 효율적으로 다듬는 역할 |
| `everything-claude-code` | **프로젝트 규칙집** — 이 프로젝트에서 코드를 어떻게 작성해야 하는지 규칙을 알려주는 역할 |
| `skill-maker` | **스킬 공장** — 한 문장만 알려주면 질문하고 설계해서 새 스킬을 만들어주는 역할 |
| `production-design` | **수석 디자이너** — 인터페이스를 프로덕션 수준으로 다듬고, AI가 만든 느낌을 제거해주는 역할 |
| `animation-engineering` | **모션 전문가** — 애니메이션과 인터랙션을 물리 법칙처럼 자연스럽게 만들어주는 역할 |
| `design-quality` | **디자인 감별사** — AI 특유의 제네릭 패턴을 감지하고 프리미엄 품질로 교정해주는 역할 |

### Step 4: 실행

사용자가 확인하면:

1. 매칭된 스킬을 순서대로 호출합니다
2. 여러 스킬이 필요한 경우, 의존관계를 고려하여 순차 또는 병렬로 실행합니다
3. 각 스킬의 결과를 통합하여 최종 결과를 제시합니다

## Query 재작성 가이드라인

- **구체화**: "예쁘게 만들어줘" → "ui-ux-pro-max 스킬의 글래스모피즘 스타일 가이드를 적용하여 카드 컴포넌트를 리디자인"
- **스킬 특화**: "발표자료 만들어줘" → "slides 스킬의 Chart.js 통합과 디자인 토큰을 활용한 HTML 프레젠테이션 생성"
- **조합 활용**: "미래에셋 대시보드" → "mirae-asset-design의 컬러 시스템 + ui-styling의 shadcn 컴포넌트 + ui-ux-pro-max의 대시보드 레이아웃 패턴"
- **최대 활용**: 단일 스킬로 충분해도 보조 스킬의 참조 자료가 품질을 높일 수 있으면 포함

## 매칭 규칙

1. **정확 매칭 우선**: 키워드가 직접 매칭되면 해당 스킬 선택
2. **의미 매칭**: 직접 키워드가 없어도 의도가 스킬 범위에 해당하면 매칭
3. **복합 매칭**: 하나의 지시가 여러 스킬에 걸치면 모두 포함
4. **매칭 없음**: 어떤 스킬과도 매칭되지 않으면, 스킬 없이 직접 처리한다고 안내

## 예시

**입력**: "미래에셋 스타일로 포트폴리오 리포트 페이지 만들어줘"

**출력**:
```
📋 스킬 매칭 결과

원본 지시: 미래에셋 스타일로 포트폴리오 리포트 페이지 만들어줘

🔧 사용할 스킬:

  1순위: mirae-asset-design
     🎯 하는 일: 미래에셋 브랜드 가이드북 역할입니다.
        미래에셋 공식 오렌지/블루 색상, Spoqa Han Sans 글꼴, 로고 배치 규정 등을
        자동으로 적용해서 "미래에셋답게" 보이도록 만들어줍니다.
     📌 왜 필요한가: 미래에셋 스타일을 요청하셨으므로 공식 가이드라인이 필수입니다.

  2순위: ui-styling
     🎯 하는 일: 가구 제작자 역할입니다.
        리포트에 필요한 표(테이블), 카드, 수치 표시 등 실제 화면 부품을
        깔끔하게 만들어줍니다.
     📌 왜 필요한가: 리포트 페이지는 데이터를 보여주는 표와 카드가 핵심이기 때문입니다.

  3순위: ui-ux-pro-max
     🎯 하는 일: 인테리어 디자이너 역할입니다.
        금융 리포트에 적합한 차트 종류 선택, 정보 배치, 한눈에 보이는
        레이아웃 구성을 안내합니다.
     📌 왜 필요한가: 금융 데이터를 효과적으로 보여주는 차트와 레이아웃 패턴이 필요합니다.

🤝 스킬 협업 방식:
  mirae-asset-design이 "미래에셋 브랜드 규격"을 정하면,
  ui-ux-pro-max가 "리포트의 전체 구조와 차트 배치"를 설계하고,
  ui-styling이 "실제 화면 부품"을 조립합니다.

📝 재작성된 Query:
  mirae-asset-design 가이드라인의 컬러 시스템과 리포트 포맷을 기반으로,
  shadcn/ui 테이블/카드 컴포넌트를 사용하여 포트폴리오 리포트 페이지를
  구현해주세요. 차트는 ui-ux-pro-max의 금융 차트 가이드를 참고합니다.

이대로 진행할까요? (Y/수정사항)
```

## Notes

- 이 커맨드는 scaffold입니다. 새 스킬이 추가되면 Step 1의 카탈로그를 업데이트하세요.
- 스킬 확인 단계를 **절대 건너뛰지 마세요**. 사용자의 의도와 다른 스킬이 매칭될 수 있습니다.
- 외부 스킬(claude-api, deploy-to-vercel 등)은 Skill 도구로 호출하고, 내부 스킬(skills/ 디렉토리)은 해당 SKILL.md의 가이드를 따릅니다.
