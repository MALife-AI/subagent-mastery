# /skill-search

자연어로 검색하면 **사용 가능한 스킬 중 관련된 것들을 찾아서** 각 스킬이 무엇을 할 수 있는지 알려줍니다.

## Goal

스킬 이름을 몰라도 "이런 거 할 수 있어?" 라고 물으면 관련 스킬을 찾아주는 검색 도구입니다.

## 실행 순서

### Step 1: 검색어 분석

사용자의 검색어에서 다음을 추출합니다:

- **작업 유형**: 만들기, 리뷰, 최적화, 배포 등
- **대상**: UI, 배너, 슬라이드, 코드, API 등
- **기술 스택**: React, Next.js, Tailwind, Flutter 등
- **스타일/도메인**: 미래에셋, 글래스모피즘, 금융, 모바일 등

### Step 2: 스킬 카탈로그 매칭

아래 전체 카탈로그에서 검색어와 관련된 스킬을 찾습니다.

### Step 3: 결과 출력

매칭된 스킬을 **관련도 순**으로 다음 형식으로 출력합니다.
사용자가 비개발자일 수 있으므로, 각 스킬이 **실제로 무슨 일을 하는지 비유나 쉬운 말로** 설명합니다.
기술 용어를 쓸 때는 반드시 괄호 안에 쉬운 설명을 병기합니다.

```
🔍 "{검색어}" 관련 스킬 검색 결과

━━━ 높은 관련도 ━━━

  📦 {스킬명}
     한마디로: {비개발자도 바로 이해할 수 있는 비유 한 문장}
     할 수 있는 것: {구체적 기능을 쉬운 말로 나열}
     사용법: /{스킬명} {argument-hint}

  📦 {스킬명}
     ...

━━━ 참고할 만한 스킬 ━━━

  📎 {스킬명} — {왜 관련될 수 있는지 쉬운 말로 한 줄}

━━━━━━━━━━━━━━━━━━━━━

총 {N}개 스킬 발견
💡 바로 실행하려면: /smart-skill {원본 검색어를 작업 지시로 변환한 문장}
```

### Step 4: 추가 탐색 제안

검색 결과가 0건이면:
- 유사한 키워드로 재검색을 제안합니다
- 전체 스킬 목록 보기를 안내합니다

검색 결과가 있으면:
- `/smart-skill`로 바로 실행할 수 있는 예시 명령을 제안합니다
- 관련 스킬들의 조합 활용법을 짧게 안내합니다

## 검색 매칭 규칙

1. **키워드 직접 매칭**: 검색어가 스킬의 키워드에 직접 포함 → 높은 관련도
2. **의미 매칭**: 검색어의 의도가 스킬의 기능 범위에 해당 → 높은 관련도
3. **간접 매칭**: 검색어의 작업에 보조적으로 활용 가능 → 참고할 만한 스킬
4. **스택 매칭**: 기술 스택 이름이 매칭 → 해당 스택 전용 스킬 우선
