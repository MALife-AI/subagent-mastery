Q.

그게 "터미널"인가요?

왜 멀쩡한 마우스 놔두고 굳이 저기다 글자를 치시는 거예요?
A. 컴퓨터를 다루는 두 가지 방식

컴퓨터에게 일을 시키는 방식은
크게 두 가지가 있습니다.

평소 우리가 쓰는 방식은 GUI(그래픽 방식) 입니다.
폴더 아이콘 더블클릭, 우클릭, 드래그 앤 드롭.

터미널은 같은 일을 글자(명령어)로 시키는 CLI 방식입니다.

GUI (Graphical User Interface) = 마우스로 클릭·드래그하는 방식

CLI (Command Line Interface) = 키보드로 명령어를 입력하는 방식
→ 그 CLI를 위한 창이 바로 터미널(Terminal)입니다.

Why Terminal?

왜 멀쩡한 마우스 놔두고
터미널에 치는 건가요?

CLI가 더 편한 상황이 분명히 있기 때문입니다.

1

반복·대량 작업에 압도적으로 빠름

파일 1,000개의 이름을 일괄 변경한다고 해보면, 마우스로는 하루 종일 걸리지만 터미널은 명령어 한 줄이면 끝납니다.
엑셀에서 셀 하나씩 고치는 것과 함수 하나로 1만 행을 처리하는 것의 차이와 같습니다.

2

자동화가 가능함

"매일 아침 9시에 이 폴더 백업" 같은 작업을 명령어 묶음(스크립트) 으로 만들어 자동으로 돌릴 수 있습니다. GUI로는 사람이 매번 클릭해야 합니다.

3

서버·클라우드에는 아예 화면이 없음

인터넷 서비스의 뒷단 컴퓨터(서버)는 대부분 화면 자체가 없습니다. 마우스·모니터 없이 원격으로 글자만 주고받기 때문에, 터미널 외에 다른 선택지가 없습니다.

4

프로그래밍용 AI가 터미널에서 돌아감

Claude Code, Cursor 같은 도구는 터미널에서 명령어로 실행합니다. AI가 폴더를 뒤지고, 파일을 고치고, 결과를 확인하려면 CLI 환경이 필수이기 때문입니다.

In Insurance Terms

보험업 실무에 빗대어 보면

GUI
한 건씩 직접 처리

심사 담당자가 청약서 한 건 한 건 화면에서 보고 승인 처리하는 방식.

한 건씩 꼼꼼히 볼 때는 직관적이고 편함.
단, 건수가 많아지면 시간이 기하급수적으로 증가.
CLI (터미널)
배치로 묶어 일괄 처리

같은 일을 배치(batch)로 묶어 야간에 일괄 처리하는 방식.

10만 건을 한 번에 돌려야 할 때 압도적으로 효율적.
터미널 = 그 "일괄 처리 창구" 라고 보시면 됩니다.

One-line Summary
한 줄 정리
GUI = 마우스로 클릭 (탐색기, Finder)
CLI = 글자로 명령 (터미널)
터미널은 반복 작업 · 자동화 · 서버 작업에 유리하고,
프로그래밍용 AI 도구의 기본 환경입니다.
